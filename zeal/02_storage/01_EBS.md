# EBS

Suppose we have a external hard disk 
We can attach the hard disk or detach the hard disk to the laptop or computer
We can detach and attach to other computer aswell
Even if the computer gets spoilt, the data stored on hard disk is safe

Similarly
- EBS is elastic block storage
- EBS volume is attached to an EC2 instance
- EBS can be configured to remain even after ec2 instanced is terminated
- We can detach and attach the volume from/to an ec2 instance
- the data stored on ebs volume is safe.
- EBS is attached to the EC2 instance via the network
- It is a network mount
- We can attach ebs volume to ec2 insstance within same AZ
- It is a AZ specific service
- We cannot attach ebs across the availabiluity zone
- We can attach nmultiple ebs to an ec2 instance
